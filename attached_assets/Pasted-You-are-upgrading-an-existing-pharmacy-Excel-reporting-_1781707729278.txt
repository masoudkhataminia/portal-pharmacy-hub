You are upgrading an existing pharmacy Excel reporting web app.

This is an upgrade, not a rebuild.

CRITICAL RULES:

1. Do not rebuild the whole app.
2. Do not redesign existing working parts.
3. Do not remove existing functionality.
4. Do not change the current patient-block Excel report layout unless explicitly required below.
5. Do not create unnecessary dashboards or extra sheets by default.
6. Do not combine multiple medications into one cell.
7. Every medication must stay in its own separate row.
8. Only add the requested intelligent review, correction, saving, and analytics features.
9. The final Excel file must open in Microsoft Excel without repair warnings.
10. Accuracy and editability are more important than decoration.

The app must become an intelligent pharmacy finance reconciliation tool, not only a static Excel generator.

OVERALL WORKFLOW:

The app should work like this:

1. Upload files
2. Analyse and match data
3. Show interactive review and correction screen
4. Allow user to fix/edit/confirm/exclude records
5. Allow user to select optional analytics/report sections
6. Generate final Excel report
7. Allow saving/finalizing the report for that month

Do not jump directly from upload to final Excel.

The correct flow is:

Upload
→ Analyse & Match
→ Review / Edit / Confirm
→ Select report sections
→ Generate Excel
→ Save finalized report

PATIENT MATCHING:

Use intelligent AI-assisted patient matching.

Monthly report:

* Patient Name is one full-name column

Scripts report:

* Patient First Name
* Patient Last Name

Build script full name as:
Patient First Name + " " + Patient Last Name

Before matching, normalize names:

* lowercase
* trim spaces
* collapse multiple spaces
* remove text inside parentheses
* remove commas and full stops
* preserve hyphenated names
* handle reversed name order
* handle middle names
* handle missing middle names
* handle initials
* handle minor spelling differences
* handle capitalization differences

Examples:
Marylou Rivas-Slone must match Marylou + Rivas-Slone (RDH)
Nanette Fairall must match Nanette + Fairall
Reginald Mcnamara must match Reginald + Mcnamara
Clinton Dixon may match Clinton Ronson Dixon, but mark review required unless confidence is very high.

Do not require DOB.
Do not block matching because DOB is missing.
Do not use HRN / Patient ID as primary matching unless proven reliable.

Matching order:

1. Exact normalized full-name match
2. AI-assisted intelligent name matching
3. Fuzzy matching backup
4. If no reliable match, mark as NO SCRIPT NAME FOUND IN SOURCE FILE

Every match must have:

* Match Status
* Match Method
* Match Confidence
* Review Notes

Confidence guide:
100 = exact normalized full-name match
95–99 = AI-assisted very strong match
85–94 = strong AI/fuzzy match, review recommended
75–84 = possible match, needs review
below 75 = do not auto-match

Do not silently accept uncertain matches.

DATA EXCLUSIONS BEFORE MATCHING:

Before matching, calculation, counting, or reporting, exclude:

1. Administrative discharge placeholder rows:
   Remove any Drug Description containing:

* RDH DISCHARGE
* PRH DISCHARGE
* ASH DISCHARGE
* equivalent non-medication discharge placeholders

2. Deferred rows:
   Only use the actual column named Deferred.
   Exclude if Deferred = Yes / Y / TRUE / 1.

Do not use CTG Script or Electronic Prescription as Deferred.

Do not remove scripts by month/date.
Do not remove “other-month” records.
If a dispense date is outside the expected month, keep it and add a note.
